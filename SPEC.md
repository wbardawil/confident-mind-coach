# Product Specification

## Purpose

The application helps users build confidence through structured mental habits.

Confidence is treated as a skill developed through daily reflection, preparation, and learning from experience.

## Key Workflows

### Onboarding

Collect:
- role
- performance domain
- strengths
- main confidence challenges
- recurring triggers
- baseline confidence score

### Top Ten

Users store ten meaningful accomplishments.

Purpose:
- create durable confidence memory
- reinforce identity
- provide material for affirmations

### Daily ESP

Daily reflection:
- Effort
- Success
- Progress

Results:
- journal entry stored
- AI coaching generated
- affirmation created
- ledger deposit added

### Pregame Routine

Input:
- upcoming event
- confidence level
- fear
- definition of success

Output:
- Take Stock summary
- Situation assessment
- Decide You Are Enough statement
- visualization prompt

### Reset Routine

Input:
- event description
- emotional state
- confidence score

Output:
- acknowledgement
- safeguard application
- next action cue

### AAR

After Action Review.

Input:
- what happened
- so what
- now what

Output:
- lessons learned
- improvement plan
